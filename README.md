# Text Analysis of Quran

Python project for thematic and exploratory analysis of English Quran translations.

## Quickstart

1. Create and activate a virtual environment.
2. Install dependencies:

```bash
pip install -e .
```

3. Copy environment template:

```bash
cp .env.example .env
```

4. Fetch chapter metadata from Quran Foundation API:

```bash
python scripts/fetch_chapters.py
```

5. Build normalized verse dataset for one English translation:

```bash
python scripts/build_translation_dataset.py
```

This writes:

- `data/raw/verses_translation_<id>.json`
- `data/processed/verses_translation_<id>.csv`
- `data/processed/verses_translation_<id>.parquet`

Set translation ID in `.env` via `QURAN_TRANSLATION_ID`.

## Exploratory Analysis

After building the dataset, the EDA lives in [`notebooks/01_eda.ipynb`](notebooks/01_eda.ipynb),
a thin presentation layer over reusable, plot-free helpers in the package:

- `quran_analysis.text` — cleaning (footnote/markup stripping), an accent-folding
  tokenizer, and two text variants: `clean_text` (keeps translator `[interpolations]`)
  and `scripture_only` (drops them).
- `quran_analysis.eda` — data-quality, structural, length, lexical, ordering, and
  theme helpers (all returning DataFrames/Series).
- `quran_analysis.themes` — curated keyword lexicons for 12 themes plus motif scoring.

The notebook is generated from [`scripts/_build_notebook.py`](scripts/_build_notebook.py)
so it stays reproducible from source; regenerate and execute with:

```bash
python scripts/_build_notebook.py
jupyter nbconvert --to notebook --execute --inplace notebooks/01_eda.ipynb
```

Sections covered: **Data Quality**, **Structural Overview** (ordering: canonical
vs chronological, ring-structure test; Makkah/Madinah profile), **Text Length**,
**Lexical** (Zipf, per-chapter signature terms, size-matched vocabulary richness),
and **Themes** (curated lexicons, length-controlled Makkah/Madinah split).

> All findings describe properties of the **English translation**, not the Arabic
> source text.

## Initial Project Layout

- `src/quran_analysis/`: package code
- `scripts/`: runnable scripts for data ingestion and utilities
- `data/raw/`: unprocessed downloaded data
- `data/processed/`: cleaned/tabular outputs
- `notebooks/`: analysis notebooks (generated by `scripts/_build_notebook.py`)

## Data Source Notes
